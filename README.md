# Transparency Estate — карта объектов

Внутренний инструмент: интерактивная карта объектов проектирования и стройки.
Каждая точка — объект со статусом, лентой работ, фото, ответственным и сроком.
Карта на Яндексе, бэкенд на Supabase. Без сборки — один HTML + статика.

## Что умеет

- **Карта** объектов (Яндекс), маркеры покрашены по статусу, кластеризация при увеличении масштаба
- **Вход** по почте (magic link). Телефон/Telegram — позже
- **Объекты**: добавить (клик по карте → форма), редактировать, удалить, обложка
- **Лента работ** внутри объекта: текст + статус + несколько фото, таймлайн, лайтбокс
- **Поля объекта**: адрес, ответственный (кнопка «позвонить»), плановый срок (подсветка просрочки)
- **Поиск** и **фильтр по статусу** в сайдбаре
- **Роли**: админ видит все объекты и управляет доступами; обычный пользователь — только выданные ему
- **Доступы**: админ выдаёт доступ к объекту по почте сотрудника
- **Геолокация** «где я», маршрут к объекту в Яндекс.Навигатор и 2ГИС
- **PWA**: ставится иконкой на телефон, оболочка работает офлайн

## ⚠️ Чтобы заработали запись и роли — залить миграцию

`schema.sql` уже в базе (демо-объекты видны). Но **добавление/редактирование объектов,
загрузка фото и роли** требуют второй миграции:

1. Supabase → SQL Editor → New query → вставить весь `schema-v2.sql` → **Run**.
   Последняя строка делает тебя (`rysky2016@gmail.com`) админом — после этого видишь все объекты.
2. Перезайти в приложении (Выйти → Войти), чтобы подхватилась роль.

Без этого шага вход и просмотр работают, а кнопки «Добавить/Сохранить/Загрузить фото» будут давать ошибку прав.

## Запуск локально

```
cd "Kampika/map-app"
python3 -m http.server 8077
```
Открыть `http://localhost:8077`. В Supabase → Authentication → URL Configuration
должен быть добавлен `http://localhost:8077` (Site URL и Redirect URLs).

## Ключи

- Яндекс JS API key и Supabase **публичный** ключ зашиты в `index.html` — это нормально, они клиентские.
- **Секретный** ключ Supabase (`sb_secret_...`) в приложении не используется и не должен. Если он где-то засветился — отозвать в Project Settings → API Keys.

## Что осталось на потом (по приоритету)

1. **Вход по телефону** — подключить SMS-провайдер в Supabase (Twilio и т.п.), затем включить phone OTP.
2. **Вход через Telegram** — Login Widget + проверка подписи токеном бота (~30 строк, нужен бот через @BotFather и домен).
3. **Хостинг** — выложить на Vercel/Netlify (статика), привязать домен, в Яндексе привязать API-ключ к домену.
4. **Приватность фото** — сейчас бакет публичный; при необходимости перейти на signed URLs.
5. **Уведомления** при смене статуса — Supabase Edge Function + почта/телеграм.

## Архитектурные решения (коротко)

- Один HTML без сборки — чтобы запускалось одной командой и правилось без тулчейна.
- Права доступа — это **RLS-политики в базе**, не код приложения. Клиент не может обойти их даже теоретически.
- Триггеры в БД: автодоступ создателю объекта, бамп `updated_at` при новой записи в ленте, профиль при регистрации.
